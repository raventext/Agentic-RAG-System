ABSTRACT
Vacuity, or Uncertainty Mass (UM), is commonly used as a metric to evaluate Out-of-Distribution
(OOD) detection in Evidential Deep Learning (EDL). It generally involves dividing the number of
classes (K) by the total strength of belief (S) of the model’s predictions, where S is derived from
summing the Dirichlet parameters. As such, UM is sensitive to the cardinality of K. In particular, it
is unlikely in practice that there is a linear relationship between K and S as K and S increase due
to the nature of EDL (suppressing incorrectly assigned evidence). As a result, when comparing In
Distribution (ID) and OOD results, it is important that KID and KOOD are equal; something that is
not always ensured in practice. We provide an empirical demonstration of how results for AUROC
and AUPR can substantially differ when class cardinality between ID and OOD differs by 1, with
AUROC differing by as much as 0.318 and AUPR by 0.613 for standard EDL, and AUROC by 0.360
and AUPR by 0.683 for IB-EDL. More concretely, our findings isolate an evaluation artefact: when K
differs between ID and OOD, AUROC/AUPR can be artificially inflated without any change in model
predictions. We further discuss the evaluation of EDL over causal language models using Multiple-
Choice Question-Answer (MCQA) datasets and argue for clearer definitions of ID and OOD in this
context. Our primary contribution is an empirical and theoretical demonstration that vacuity-based
OOD detection in EDL-fine-tuned LLMs is highly sensitive to uncontrolled differences in evaluated
class cardinality. Code Available:

Introduction
Evidential Deep Learning (EDL) [1] has emerged as a promising way to quantify uncertainty in causal large language
models (LLMs) by fine-tuning them to operate as next token classifiers over Multiple-Choice Question-Answer (MCQA)
datasets [2, 3]. The utilisation of datasets like MCQAs rather than free-text generation stems from how EDL operates.
More specifically, it requires a set number of classes with which to create Dirichlet parameters over. Summing the
Dirichlet parameters gives you the model’s total strength of belief (S) and from this, vacuity can then be understood
as the uncertainty mass, K
S . It is important to note that in EDL, uncertainty mass (UM) is a quantitative measure,
vacuity is its interpretation as ignorance-based uncertainty, and epistemic uncertainty is the broader concept that vacuity
approximates but they are not strictly identical. For the purpose of this paper however, when we refer to vacuity we
mean UM. While calculating vacuity-based uncertainty in this manner can be useful [1, 4, 5], as we will present in this
paper, i) vacuity-based Out-of-Distribution (OOD) detection can be highly sensitive to class number (K) specification
in MCQA-based EDL settings, and (ii) as such, due to this sensitivity, when using it to compare results across datasets,
great care must be taken to ensure that the value of K remains constant. This issue is particularly salient in MCQA-based
LLM evaluation because the class set is not a fixed semantic label space in the same way as standard image classification
(where EDL originates [1]). The labels, such as A - D, are placeholders whose semantic content changes with each
question, and the number of answer options can differ across datasets. Therefore, class-dependent uncertainty quantities
may reflect changes in task format or answer-option structure rather than only epistemic uncertainty about the input.

Specifically, the paper highlights an implementation-sensitive failure mode in vacuity-based OOD detection for MCQA
fine-tuned LLMs, rather than proposing a new uncertainty method. As such, this paper demonstrates that vacuity-based
OOD detection in EDL-fine-tuned LLMs can be highly sensitive to uncontrolled differences in effective class cardinality
(K), leading to substantially inflated AUROC and AUPR estimates.
This paper makes three contributions:
1. We empirically demonstrate that mismatched effective class cardinality can substantially inflate AUROC and
AUPR for vacuity-based OOD detection in EDL-fine-tuned LLMs.
2. We show formally that vacuity, K
S , is not invariant to class expansion unless the added class receives concen-
tration equal to the mean concentration of the original classes.
3. We discuss why MCQA-based LLM evaluation differs from fixed-label classification and why clearer defini-

The paper first introduces EDL and a variation of it more formally, it then presents how vacuity-based uncertainty is
used currently to detect OOD inputs, next it shows the impact of (i) and (ii) above in practice, and finally, the impact of
the current utilisation of uncertainty within the context of LLMs and the broader EDL domain is discussed.
Background and Related Work
As mentioned in the introduction, in order to work around the fact that causal language models operate by outputting
tokens sequentially rather than performing classification tasks, it is common to fine-tune an LLM using Parameter
Efficient Fine-Tuning (PEFT) methods like LoRA [6] on MCQA datasets when examining how they perform with EDL
[2, 3]. Working this way yields a constrained set of possible next token options - the options (classes) of the MCQA
dataset. This section details how EDL operates over MCQA datasets with LLMs and outlines a variation of EDL -
Information-Bottleneck EDL (IB-EDL) [2].

2.3
Metrics Used in Evaluation
There are a variety of methods used to assess a fine-tuned LLMs performance when fine-tuned using EDL or one of
its variations. One of the most commonly used ones is Expected Calibration Error (ECE) [7, 2, 8]. Another method
used to assess model performance in general across modalities (Computer Vision, CV, for instance) is an assessment of
how well the model performs when conducting Out-of-Distribution (OOD) detection. This is typically carried out by
using the Area Under the Receiver Operating Characteristic curve (AUROC) [9] or Area Under the Precision-Recall
curve (AUPR) [10] using metrics like UM (vacuity) with ID and OOD inputs [2, 5, 11]. We raise concerns over
evaluating model performance using OOD with UM in this manner as will be presented below. Broader skepticism
over vacuity-based uncertainty measured as K
S has been raised describing the formulation as "somewhat arbitrary" [12]
noting that it does not follow from subjective logic and may not achieve practical effectiveness.

Impact in Practice
This section details how using different numbers of K can impact results in practice using a published account.
4.1
OOD using UM and MP
A motivating example of the unreliability of using vacuity-based uncertainty as a means of detecting OOD inputs in
practice can be seen through a faithful reproduction of the work of the authors of IB-EDL [2]. As discussed above,
Rethinking Vacuity for OOD Detection in Evidential Deep Learning
IB-EDL aims to reduce spurious evidence by applying a bottleneck regularisation over the evidence. The results show
that when Llama3-8b [15] is fine-tuned on variations of EDL and equivalent methods (e.g., R-EDL [11] and I-EDL
[16], IB-EDL generally maintains or increases accuracy across the MCQA datasets tests and reports generally lower
ECE results. Thus it suggests that the IB-EDL does indeed generally work better in practice when calibrating an LLM.
An important consideration that arose during reproduction however was that OOD detection was achieved using the
reciprocal of Uncertainty Mass (UM) and Max Probability (MP) with ID=1 and OOD=0. AUROC results from [2]
report that both using the UM and MP results in strong OOD detection across EDL methods, models, and datasets
indicating that both metrics are useful when used in the context of detecting OOD inputs. Specifically, the reported
AUROC for reciprocal of UM across all tested MCQA datasets for Llama3-8b with IB-EDL was reported to be between
85.45 and 94.77, and the MP was reported to be between 88.14 and 89.16. During reproduction, while it was possible
to achieve close to the reported number using the author’s provided GitHub implementation 3, it was not possible to
recreate close to the reported numbers for UM nor MP using an independent implementation. A detailed account of the
full reproduction procedure can be found in the Appendix 7.2. This prompted an inspection of the released dataset
helper code that indicated that for all OOD tests in the main paper, while the number of classes (K) for the ID dataset
was four, the number of classes for the OOD datasets tested was five. The result was a much larger difference between
ID and OOD uncertainties and probabilities. This highlighted two issues: 1., vacuity-based uncertainty is sensitive to K
(naturally as K
S ) more specifically, when class cardinality differs, the impact is a much larger separation between ID
and OOD in AUROC; from which the second point directly follows, 2., when corrected, UM and MP perform weakly at
detecting OOD in this context. It is important to note that we include MP for completeness in line with IB-EDl [2] but
is not governed by the same K
S dependence as vacuity and is not a primary object of analysis.

As mentioned previously, implementation B is our separate implementation of fine-tuning Llama3-8b with IB-EDL and
standard EDL on the OBQA MCQA dataset using the same setup (hyperparameters, training steps, etc.) as [2] and

A core difference between implementations A and B is the fact that the predictions for implementation B with ARC-C

task-format shift when evaluating OOD detection in MCQA-based EDL, something which is discussed in more detail
below.

Discussion
We do not argue that EDL or IB-EDL is ineffective as a calibration method and these findings as such should be
interpreted as a clarification of evaluation conditions rather than a critique of prior work. When class cardinality
is controlled, IB-EDL continues to exhibit strong calibration properties; however, OOD separability measured via
class-dependent metrics requires careful interpretation. Indeed, our reproduced fine-tuning results show competitive
accuracy, ECE, and NLL. Our claim is narrower: vacuity-based OOD comparisons can be misleading when the effective
number of evaluated classes differs between ID and OOD datasets. This phenomenon is not unique to IB-EDL. For
example, [8] evaluate a model trained on a 2-class dataset (Amazon Reviews [20]) on a 10-class dataset (Yahoo Answers
[21]). While their work does not rely on vacuity, their uncertainty measures (predictive entropy, expected entropy, and
mutual information) are defined over the output simplex and therefore scale with the number of classes K. As such,
differences in uncertainty between ID and OOD settings may also reflect differences in output dimensionality rather
than solely differences in data distribution. More generally, this highlights that evidential uncertainty measures are
defined with respect to a particular output space. When the dimensionality or semantics of this space change between
ID and OOD settings, the resulting uncertainty values are not strictly comparable, even when computed correctly. In
practice, this suggests that controlling for class cardinality, or applying appropriate normalisation, is important when
interpreting uncertainty-based OOD comparisons.
The consideration of what metric to use when detecting OOD inputs raises a larger issue relating to the ambiguity of
what constitutes OOD input in the context of an EDL evaluation with LLMs. In the context of an LLM fine-tuned
Rethinking Vacuity for OOD Detection in Evidential Deep Learning
on common-sense MCQA data, should we consider a different MCQA common-sense dataset truly OOD? While it
may not reflect the precise data the model was trained on, it likely can still be considered within the realm of its ID
distribution. In other words, it may be more apt to consider such datasets as near OOD [18] datasets. As seen above,
when confronted with input data from a different domain to its training data (Math MMLU), MP and UM perform better
as OOD detectors. Such datasets perhaps can be considered as far OOD [18] ones. Furthermore, due to the nature of
LLMs and the method by which they are fine-tuned, there is another distinction we can make - task OOD. If considering
an LLM that has been fine-tuned to distinguish between four options of an MCQA input, can we consider providing it
with five options instead as an alternate source of OOD?
A related, though not identical, consideration arises in CV classification scenarios with EDL, the domain from which it
originates. In standard CV OOD evaluation, the classifier output space usually remains fixed by the ID training labels;
for example, a CIFAR-10 classifier still produces CIFAR-10 outputs when evaluated on CIFAR-100 images. Therefore,
the explicit MCQA class-cardinality mechanism demonstrated here does not directly transfer to CV. However, methods
such as R-EDL [11], while reducing the explicit dependence of UM on the number of classes through a learnable prior
weight, may still raise broader questions about how vacuity-based scores vary across OOD datasets with different
semantic granularity. Consequently, comparisons across heterogeneous OOD datasets may still risk conflating epistemic
uncertainty with differences in semantic granularity, domain shift, or task mismatch. A related consideration also arises
for I-EDL [16]: although it does not explicitly use UM, it relies on the total evidence strength S, which can vary with
the model’s evidence allocation over its evaluated output space. Thus, while the explicit class-cardinality mismatch
demonstrated in MCQA is less applicable to standard CV classification because the output space remains fixed by the
training labels, care is still needed when interpreting vacuity- or evidence-based OOD scores across heterogeneous
OOD benchmarks. Shen et al. [22] suggest that the current definitions of epistemic and aleatoric uncertainty and the
way in which they are measured within context of EDL is misaligned due to the absence of model uncertainty and that
while methods appear to work downstream, that is in spite of their poor ability to capture uncertainty. In other words,
EDL in its current form may reflect properties of the predicted distribution rather than fully capturing model uncertainty
in the Bayesian sense.
Reasoning over these problems and considerations surfaces an underlying assumption - that causal language models
operating over MCQAs can be considered comparable to CV models operating over image classification. The problem
is that MCQA fundamentally differs from CV classification in that the output space is dynamically defined by the input.
More specifically, in MCQA, the semantic meaning of classes is instance-dependent, unlike fixed semantic labels in
standard image classification. As a result, uncertainty or belief measures that depend on output dimensionality or class
semantics are not invariant under changes in the number of answer options, even when the underlying task remains
identical. This makes MCQA a less than ideal proxy for evaluating class-based uncertainty or causal belief models
without explicit correction. There are further concerns raised about using MCQAs in general to evaluate uncertainty
in LLMs [23] where the output may reflect less about uncertainty and more general decoding variability. Other work
has surfaced the fact that LLMs may default to choosing the "least incorrect option" [24] raising questions about their
performance when no correct option is available.
Conclusion
Our findings do not argue against evidential uncertainty itself, but against comparing vacuity-based OOD scores
across mismatched class cardinalities. We argue that when considering using vacuity as UM for OOD detection, it is
imperative that KID and KOOD are equal and demonstrate how K-variance can impact results in practice. We further
raise questions surrounding the direct transferability of using EDL in the context of causal language models over
MCQAs and a need for greater clarity in what exactly is being thought of as OOD input. It is hoped that our work will
contribute to greater transparency in how ID and OOD datasets are defined within the context of EDL with LLMs and
result in greater care being taken when calculating and comparing UM as a metric for OOD detection. Our follow-up
work aims to be able to clarify types of OOD input in this context and how they affect EDL in practice. In short,
vacuity-based OOD comparisons are only meaningful under controlled output dimensionality; otherwise, they reflect
evaluation artefacts rather than model behaviour.

Implementation Procedure
The same hardware and hyperparameters were used for both Implementation A and Implementation B. Specifically,
in order to both fine-tune the Llama3-8b and generate predictions for ID and OOD inputs, we used one A100 GPU.
Following [2], we fine-tuned Llama3-8b for 10080 steps on the OBQA dataset using LoRA with the following
hyperparameters: r= 8, alpha= 16, dropout: 0.1, targeting the query and value projections and the output head. It should
be noted that for both implementations, we obtained results similar, though not identical, to those reported by the
authors. Such variation can arise from stochastic training effects and environment-level nondeterminism common in
LLM fine-tuning. Exact replication of fine-tuning metrics was not the primary objective of this study, as our focus was
on assessing metric differences arising from class cardinality rather than differences in model training performance.

Implementation A
